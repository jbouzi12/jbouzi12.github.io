SQL INJECTION: An in Depth Look



SQL Injection is one of the many tools in a hacker's arsenal that is used to steal data from organizations. In fact, it is one of the most common web attack mechanisms today. It occurs when an incorrectly coded application allows SQL statements to pass through the available fields and query the database directly.

Databases are the bread and butter of modern websites; they store data needed for websites to send specific, private information to visitors, customers, suppliers, employees, or anyone else with a personal account on the website. This information may consist of financial and payment information, user contact information, and employee or medical records. Databases are absolutely vital for any business. SQL injection can undermine a company's internal web application structure by passing SQL commands through the application for the database to execute. This may allow hackers to view confidential information from the database (or worse, destroy it) if the code isn't sanitized properly.

Lets look at a simple example of how SQL Injection works. Say we are at a login page where a user would enter a username and password to enter a secure area to view their personal information. When this information is entered, it becomes an SQL query that is retrieved by the database for confirmation; if valid the user is allowed access. Imagine this particular website was built with database query that resembled this:

SELECT id FROM logins WHERE username = '$username' AND password = '$password'

The database can easily be compromised if $username and $password are directly requested from the user's input. For instance, we may submit 'Mike' as a username and use this as a password: 'anything' OR 'x'='x'. By using single quotes, we have rendered password to be a two-component clause. The 'x'='x' part will always be true no matter what the first part contains; this will allow us access to the database even without a valid username/password combination.

A significant number of websites are vulnerable to SQL Injection, and if hackers are unable to alter the database, they can at least view confidential information. It is difficult to suggest the best method to combat this threat due to my lack of coding experience. My research pertaining to this issue hasn's produced a solid solution. Reinforcing your servers, databases, programming languages, and operating systems will help, but won't make your website completely immune.
